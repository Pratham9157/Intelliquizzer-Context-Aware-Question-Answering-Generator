# PHASE 1 — ADVANCED PDF CLEANING

---

The current PDF extraction merges tokens incorrectly.

Examples:

* "Philipp KoehnDan Jurafsky"
* "bitextDan Jurafsky"
* "63Maria nodiounabofetada"

This breaks downstream NLP.

Implement a robust PDF/text cleaning layer.

Requirements:

1. Remove repeated slide artifacts:

* author names repeated across pages
* slide titles repeated every page
* page numbers
* isolated symbols

2. Fix merged-token artifacts:
   Examples:

* "KoehnDan" → "Koehn Dan"
* "bitextDan" → "bitext Dan"

Use regex-based spacing repair:

* detect lowercase-uppercase merges
* detect numeric-text merges

3. Remove OCR noise:
   Reject lines containing:

* excessive symbols
* corrupted unicode
* random numeric fragments
* malformed tokens

4. Remove presentation-specific noise:
   Examples:

* standalone headers
* repeated footer names
* isolated equations without explanation
* slide duplicates

5. Normalize whitespace aggressively.

Create:
preprocessing/text_cleaner.py

Functions:

* clean_extracted_text()
* remove_slide_noise()
* repair_merged_tokens()
* normalize_text()

# REQUIRED HEURISTICS

---

Reject if:

* token count < 2 OR > 8
* starts with:
  "the", "a", "an"
* contains repeated words
* contains excessive punctuation
* contains mostly uppercase/random chars
* contains assembly/code keywords:
  eax, ebx, mov, xor, ret, addr, sfp, etc.
* contains malformed OCR text
* low alphabetic ratio
* low semantic score

Reject answers if:
spaCy POS pattern lacks meaningful nouns/concepts.
